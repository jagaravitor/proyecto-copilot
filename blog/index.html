Post #1: Introducción a los Grafos

21 de Noviembre, 2025 8 min de lectura

¿Qué es un Grafo?

Un grafo es una estructura de datos que representa conexiones entre objetos. Formalmente, un grafo G = (V, E) consiste en:

  • V (Vértices o Nodos): Los objetos o puntos del grafo
  • E (Aristas o Ejes): Las conexiones entre los vértices

Conceptos Clave

Nodos (Vértices)

Son los elementos fundamentales del grafo. Pueden representar personas, ciudades, dispositivos, o cualquier entidad que queramos conectar.

Aristas (Ejes)

Son las conexiones entre nodos. Una arista conecta dos nodos y puede tener una dirección o un peso asociado.

Tipos de Grafos

1. Grafos Dirigidos (Directed Graphs)

Las aristas tienen una dirección específica. Se representan con flechas.

Ejemplo: Red de flujo de agua, relaciones de seguimiento en redes sociales

2. Grafos No Dirigidos (Undirected Graphs)

Las aristas no tienen dirección. La conexión es bidireccional.

Ejemplo: Red de amigos, red de carreteras

3. Grafos Ponderados (Weighted Graphs)

Las aristas tienen un peso o valor asociado (distancia, costo, probabilidad).

Ejemplo: Mapa con distancias entre ciudades, red con latencias

Post #2: Representación de Grafos

21 de Noviembre, 2025 10 min de lectura

¿Cómo Representar un Grafo en Memoria?

Existen dos formas principales de representar un grafo en una computadora. Cada una tiene ventajas y desventajas según el caso de uso.

1. Lista de Adyacencia

Es una colección de listas donde cada nodo tiene una lista de nodos adyacentes.

Ventajas:

  • ✓ Eficiente en espacio para grafos dispersos (pocas aristas)
  • ✓ Rápido para iterar sobre vecinos de un nodo
  • ✓ Fácil de implementar

Desventajas:

  • ✗ Consulta lenta si queremos saber si existe una arista específica
  • ✗ Más lento para grafos densos

Ejemplo de Lista de Adyacencia:

{
  'A': ['B', 'C'],
  'B': ['A', 'D'],
  'C': ['A', 'E'],
  'D': ['B'],
  'E': ['C']
}

2. Matriz de Adyacencia

Es una matriz cuadrada donde cada celda [i][j] indica si existe una arista del nodo i al nodo j.

Ventajas:

  • ✓ Consulta rápida O(1) para verificar si existe una arista
  • ✓ Fácil para buscar todas las aristas de un nodo
  • ✓ Eficiente para grafos densos (muchas aristas)

Desventajas:

  • ✗ Usa mucho espacio O(V²) incluso si hay pocas aristas
  • ✗ Ineficiente para grafos dispersos

Aspecto Lista de Adyacencia Matriz de Adyacencia

Post #3: Algoritmos de Recorrido - BFS y DFS

21 de Noviembre, 2025 12 min de lectura

¿Qué son BFS y DFS?

BFS (Búsqueda en Amplitud) y DFS (Búsqueda en Profundidad) son dos algoritmos fundamentales para recorrer o explorar todos los nodos de un grafo.

1. BFS (Breadth-First Search)

Búsqueda en Amplitud: Explora el grafo nivel por nivel, comenzando desde un nodo inicial.

Características:

  • Usa una Cola (Queue) para guardar nodos
  • Explora todos los vecinos antes de ir más profundo
  • Complejidad: O(V + E)
  • Útil para encontrar caminos más cortos

Pseudocódigo BFS:

BFS(grafo, inicio):
  cola = nueva Cola()
  visitados = conjunto vacío
  
  cola.enqueue(inicio)
  visitados.agregar(inicio)
  
  mientras cola no esté vacía:
    nodo = cola.dequeue()
    procesar(nodo)
    
    para cada vecino en adyacentes(nodo):
      si vecino no está en visitados:
        visitados.agregar(vecino)
        cola.enqueue(vecino)

Visualización BFS:

Orden de exploración: A → B → C → D → E → F → G

2. DFS (Depth-First Search)

Búsqueda en Profundidad: Explora el grafo profundamente antes de retroceder.

Características:

  • Usa una Pila (Stack) o recursión
  • Explora completamente una rama antes de pasar a otra
  • Complejidad: O(V + E)
  • Útil para detectar ciclos y ordenamiento topológico

Pseudocódigo DFS:

DFS(grafo, nodo, visitados):
  visitados.agregar(nodo)
  procesar(nodo)
  
  para cada vecino en adyacentes(nodo):
    si vecino no está en visitados:
      DFS(grafo, vecino, visitados)

Visualización DFS:

Orden de exploración: A → B → D → E → C → F → G

Comparación BFS vs DFS

Aspecto BFS DFS
Estructura Cola Pila / Recursión
Orden Nivel por nivel Profundidad primero
Complejidad O(V + E) O(V + E)
Mejor para Camino más corto Detección de ciclos
Espacio O(V) - Cola O(h) - h = altura

Casos de Uso

BFS se usa para:

  • Encontrar el camino más corto en un grafo no ponderado
  • Exploración de redes sociales (amigos más cercanos)
  • Análisis de niveles en árboles

DFS se usa para:

  • Detectar ciclos en un grafo
  • Ordenamiento topológico
  • Búsqueda de componentes conectadas
  • Resolver problemas de backtracking
